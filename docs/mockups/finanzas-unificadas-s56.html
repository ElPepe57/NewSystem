1 · El problema que resolvemos

CxC + CxP + Cuenta Corriente: 3 vistas, 1 dominio

Hoy hay 3 reportes/tabs separados (CxC, CxP, Cuentas Corrientes) que responden preguntas diferentes pero del mismo dominio: quién nos debe y a quién le debemos. El usuario se mueve entre 3 pantallas con UX inconsistente.

Después de implementar Cuenta Corriente Unificada (S55), tenemos la oportunidad de unificar la experiencia visual aplicando el modelo que el usuario ya validó como ideal en Compras y Envíos (cards densas + pipeline + detalle modal con tabs sticky).

❌ Hoy
  • 3 tabs separados sin consistencia visual
  • CxC/CxP por documento · CC por entidad → confunde
  • No hay dashboard ejecutivo único
  • Acciones rápidas (pagar/cobrar) requieren navegar a otras páginas
  • UX inconsistente con el modelo Compras/Envíos del usuario
✅ Propuesta
  • Hub único /finanzas con pipeline + cards
  • Lenguaje visual consistente con Compras/Envíos
  • Dashboard ejecutivo siempre visible (KPIs arriba)
  • Acciones rápidas inline en cada card (Cobrar / Pagar / Aplicar saldo)
  • Detalle modal con tabs (Extracto · Documentos · Histórico)
Inspiración visual: Stripe Dashboard · Linear · Mercury Banking · Notion · Plaid Dashboard. Combinada con el modelo Compras/Envios del usuario (cards densas + pipeline clickable + detalle modal con tabs).

7 · Decisiones a confirmar y plan

Decisiones de diseño (a confirmar contigo)

D-FIN-1 · Hub único en /finanzas
Crear ruta nueva /finanzas que reemplaza los 3 tabs de Reportes (CxC, CxP, Cuentas Corrientes). Aparece en sidebar como módulo independiente. Las tabs viejas se eliminan o redirigen.
D-FIN-2 · Pipeline clickable como filtro principal
5 chips visibles arriba: Todas · Por cobrar · Por pagar · Vencidas · Saldadas. Click filtra la lista. Estado activo en slate-900 (tipo Stripe). Más chips secundarios por tipo de entidad.
D-FIN-3 · Cards estilo CompraCard con layout 12-col
Cada entidad es una card con 5 zonas: (1) Avatar + nombre + tipo · (2) Estado/pipeline · (3) Saldo destacado · (4) Documentos asociados · (5) Acciones. Layout dual @container narrow/wide.
D-FIN-4 · Modal detalle estilo OrdenCompraCard
Click en card abre modal con: Header hero (saldos USD/PEN destacados) + Pipeline contextual (cuenta abierta → compras → pagos → saldada) + 4 KPIs + Tabs sticky: Extracto · Documentos · Histórico · Acciones.
D-FIN-5 · Acciones rápidas inline
Cada card tiene CTA contextual según el saldo: Cobrar (verde) si entidad nos debe · Pagar (rojo) si le debemos · Aplicar saldo (morado) si tiene saldo a favor. Abre PagoUnificadoForm ya existente.
D-FIN-6 · Mantener tabs CC en fichas Proveedor/Cliente
El acceso por contexto (entrar a la ficha de un proveedor y ver su CC) se preserva. Lo que cambia es la VISTA GLOBAL, no las vistas por contexto.
D-FIN-7 · ¿Eliminar tabs CxP/CxC viejos de /reportes?
Mi recomendación: eliminarlos del módulo Reportes y mover toda la funcionalidad a /finanzas. La vista por documento se accede via tab "Documentos" dentro del modal de detalle de la entidad. ¿O preferís dejar copia en Reportes para auditores?
Pendiente decisión.
D-FIN-8 · ¿Personas con multi-rol unificadas en una card?
Carlos puede ser cliente Y empleado a la vez (CC separadas en BD). ¿En la lista del hub mostramos 2 cards (una por rol) o 1 card con badges múltiples y saldos agregados?
Mi voto: 1 card por persona física con badges múltiples (más limpio). Pendiente confirmación.
