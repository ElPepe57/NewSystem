Recordatorio · el color de grupo NO toca el semántico

El color del grupo viste el chrome (chip · icono · tab activo · primary CTA). Los datos siguen su semántico fijo en CUALQUIER módulo:

Pagado · emerald Pendiente · amber Vencido · rose Parcial · sky
Ej: en Inventario (orange) un stock OK sigue siendo emerald, uno crítico sigue siendo rose. El orange solo está en el marco.
Cómo lo lee el sistema · registro finito (no inventa)
// grupoColor.ts · única fuente de verdad
export const GRUPO_COLOR = {
  'finanzas-contabilidad': 'teal',
  'equipo':               'violet',
  'comercial':            'blue',
  'inventario':           'orange',
  'analisis':             'indigo',
  'administracion':       'slate',
} as const;

// el Hub Kit lo resuelve solo · el módulo no decide color
<HubShell color={GRUPO_COLOR[grupoDelModulo]}> ... </HubShell>
Lo que falta decidir

teal y violet están firmes. Los 4 propuestos (Comercial=blue · Inventario=orange · Análisis=indigo · Admin=slate) son una propuesta sobria y banking-grade — ajustables. Si te cierran, queda cerrada la gobernanza de color y se documenta como canon. Las únicas tensiones (leves) son blue/indigo con semánticos poco usados; orange y slate están limpios.
