§ Intro

Arquitectura del Wizard · 1 entry point · 4 flows

El usuario entra al wizard desde el dropdown "+ Nueva cuenta" en /finanzas/cuentas. Paso 1 obliga a elegir el tipo. Cada tipo desbloquea su propio flow de pasos 2-4 con campos específicos al tipo. Color del wizard cambia según tipo (canon N4 cross-módulo).

Datos de la billetera digital

Proveedor + identificador único + comisión típica · sistema descuenta comisión automáticamente al registrar cobros.

Stripe/PayPal soportan multi-moneda · Yape/Plin solo PEN
Comisión típica del proveedor
S/
Cada cobro registrado descuenta automáticamente esta comisión y la registra como gasto en la categoría seleccionada · ej. Stripe 4.5% + S/ 0.60 · MercadoPago 4.99% · Yape 0% · Plin 0%
PASOS 3 + 4 · Billetera · estructura idéntica a Bancaria Solo cambia el tinte color → sky · default saldo 0 (billeteras se cargan al usar)
PASO 3 · Saldo inicial

Default S/ 0.00 · las billeteras digitales se cargan al usarse · contrapartida 5901 Aporte si arranca con saldo.

Casos comunes:
  • · Yape recién instalado · saldo 0
  • · Stripe con balance pendiente de transferir · saldo > 0
  • · PayPal con balance USD acumulado · saldo USD
PASO 4 · Permisos + alertas

Mismo patrón Bancaria · adicional alerta "Balance acumulado > X transferir a banco" (típico Stripe).

Alertas específicas billetera:
  • · Stripe: balance > S/ X → recordar payout
  • · Yape: límite operativo diario alcanzado
  • · PayPal: balance USD > X → considerar conversión a Pool USD

Datos de la caja efectivo

Responsable obligatorio · firma los arqueos periódicos · responsable de faltantes/sobrantes.

S/
Sistema alerta si el saldo excede este límite
Caja firmada digitalmente · cada arqueo genera un registro con firma del responsable + saldo declarado + diferencia vs. sistema. Faltantes se registran como pérdida operativa (cuenta 6593) · sobrantes como ingreso (cuenta 7593).
PASO 3 · Saldo apertura caja

Saldo de apertura (efectivo físico al crear) · contrapartida default 1041 Caja general si es traslado interno.

Casos comunes:
  • · Caja chica nueva · S/ 200 desde aporte de socios
  • · Traslado desde caja general · debita caja general · acredita caja chica
  • · Caja de tienda · saldo cero · se carga con ventas del día

§ Mobile responsive

Adaptación mobile · sidebar pasos colapsa a tabs horizontales

En viewports menores a md: (768px) · sidebar vertical de pasos se transforma en barra horizontal scrolleable (canon N6) · footer botones se apilan vertical (Crear arriba · Borrador medio · Descartar abajo discreto).

Datos de la cuenta

Cambios mobile vs desktop:
  • Sidebar pasos vertical → tabs horizontales scroll (N6)
  • Footer botones inline → apilados verticales (Crear top, Borrador medio, Descartar discreto)
  • Grid 2 columnas → 1 columna · excepto pares semánticos (Moneda + Fecha)
  • Inputs full-width · labels más chicos (10px en lugar de 11px)
  • Banners explicativos · texto reducido pero NO eliminados
