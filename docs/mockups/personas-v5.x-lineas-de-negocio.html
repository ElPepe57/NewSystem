ACTO 1 · Modelo conceptual

3 reglas que rigen el modelo

1
Empleados · multi-select

lineasNegocioIds: string[] · array de IDs de líneas. Puede tener 0 (compartido), 1 (dedicado), o N (multi-línea con UN sueldo).

Ej. Marcos ayuda en Skincare Y Maquillaje con UN sueldo S/2300 · 1 relación con array de 2 IDs.
2
Socios · single (o vacío)

lineaNegocioId: string | null · 1 valor o null. null = cap table empresa global. Equity es por entidad legal.

Ej. Juan fundador empresa = null · Pedro inversor de Maquillaje = "maquillaje". Si está en 2, 2 relaciones.
3
Sueldos separados = N relaciones

Si Marcos cobra S/1500 en Skincare Y S/800 extra en Maquillaje (sueldos distintos), se crean 2 relaciones empleado.

El admin elige conscientemente · UX: en el modal hay opción "Agregar segundo contrato con otra línea".
Por qué asimétrico: el trabajo (empleo) puede ser compartido entre líneas con UN solo contrato · el equity es siempre por entidad legal y no se "promedia". Esta asimetría refleja la realidad contable y operativa de PyMEs.

Para separar empleados o socios por línea, primero tenés que crear al menos una línea desde Configuración. Si tu negocio no tiene varias líneas, podés dejar este campo vacío sin problema.
