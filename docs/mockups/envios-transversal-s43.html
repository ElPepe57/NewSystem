📌 Cambio de modelo

El envio T1 ya no es un "paquete unico". Cuando el proveedor despacha la sub-orden en N tandas (caso Amazon con fechas 29/30-mar/01-abr), cada tanda se registra como un SubEnvioT1 con su tracking y fecha propia. El envio padre se considera "recibido completo" cuando todas sus sub-tandas estan entregadas.

🎯 Picking priorizado — detalle del algoritmo

Como el sistema sugiere que unidades incluir en un envio T2

Orden de prioridad

1
Pre-vendidas (reservada_para = cotizacion)
Cotizaciones con adelanto PAGADO. El sistema las recomienda siempre al principio del picking.
💡 Campo Unidad.reservadaPara (ya existe)
2
Stock antiguo (FIFO por fechaRecepcion)
Entre unidades no reservadas, priorizar las que llegaron primero a la casilla.
3
Stock reciente
El resto, ordenado por fechaRecepcion descendente.

Override del usuario

El sistema NO bloquea al usuario si decide no incluir pre-vendidas. Pero muestra advertencias:

⚠️
Hay 5 unidades pre-vendidas no incluidas.
Cotizaciones afectadas: COT-123, COT-145, COT-178 (adelanto pagado).
Si continuas, estos clientes seguiran esperando.
Caso edge: si la casilla no tiene NINGUNA unidad pre-vendida, el wizard salta el banner y ordena solo por FIFO. El picker se siente natural, sin ruido innecesario.
