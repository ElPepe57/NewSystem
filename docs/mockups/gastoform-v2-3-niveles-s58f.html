Manifesto · qué se gana con los 3 niveles

El modelo legacy mezclaba "tipos de gasto" en una sola lista plana (GV/GD/GA/GO). El nuevo modelo separa quién paga el costo (3 cajas fijas) de qué cosa específica es (árbol dinámico).

Modelo legacy (deprecado)

// gasto.types.ts línea 307
@deprecated categoria: 'GV' | 'GD' | 'GA' | 'GO'
  • • Sigla criptica · GV/GD/GA/GO no es legible
  • • Lista plana sin jerarquía
  • • No editable desde UI · cambios requieren deploy
  • • No diferencia "afecta CTRU" vs "no afecta"
  • • Reportes no pueden agrupar por bloque
✓

Modelo nuevo (3 niveles)

// gasto.types.ts línea 312
categoriaCostoId?: string
// → CategoriaCosto.bloque + categoriaPadreId + nivel
  • • 3 bloques fijos con semántica de negocio clara
  • • Árbol jerárquico padre/hijo
  • • Editable desde Maestros · sin deploy
  • • Bloque define automáticamente impacto en CTRU
  • • Reportes BI por bloque · categoría · subcategoría
