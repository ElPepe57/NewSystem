Hub Kit · la spec para construir el shell canónico (L5)

Esta es la spec de implementación de los 6 building blocks. A diferencia del blueprint conceptual (hub-design-system-v1), aquí el kit está ensamblado con contenido real, mostrando Layout A y B, la herencia de color por grupo, los estados del body y el mobile 375px. Sirve de referencia pixel-perfect para escribir HubShell / HubTopBar / HubHeader / HubKpiStrip / HubTabs / HubBody.

Radios (tokens): shell 2xl · cards xl · controles lg · pills full. Hijo nunca más redondo que el padre.
Color: chrome = color del grupo (heredado · grupoColor.ts) · datos = paleta semántica fija.
Sombra: shell shadow-sm (reposo) · overlays shadow-lg. El body va dentro del shell (1 recuadro).

ACTO 4 · Los 6 blocks · API + tokens que consume cada uno
<HubShell>
Card contenedor · 1 recuadro continuo. Envuelve todo.
<HubShell>{children}</HubShell>
tokens: radius.shell (2xl) · elevation.resting (shadow-sm) · border.default · overflow-hidden
<HubTopBar>
Breadcrumb S9.D1 dinámico + chip de rol contextual.
<HubTopBar grupo="equipo"
  modulo="Usuarios" leaf={tab}
  onVolver={...} esAdmin />
color: chromeDe(grupo).chip · text.meta · ChevronRight w-3
<HubHeader>
Icono tonal + h1 + subtítulo + acciones 3-tier (N10).
<HubHeader icon={Wallet}
  grupo="finanzas-contabilidad"
  titulo="Finanzas" subtitulo="…"
  acciones={[…]} />
icono: chromeDe(g).headerIcon · primary: .primaryBtn · text.hero (h1)
<HubKpiStrip>
KPIs semánticos (N1+N2) + mini-stats footer (N3).
<HubKpiStrip kpis={[
  {label,valor,tono:'amber',icon},…
]} miniStats={[…]} />
color SEMÁNTICO (no de grupo) · radius.shell en cada card · text.hero + overline
<HubTabs>
Tabs border-b-2 · scroll-x mobile (N6) · badge opcional.
<HubTabs grupo="…" activa={t}
  tabs={[{id,label,icon,badge?}]}
  onChange={…} />
activa: chromeDe(g).tabActive · badge = semántico · 1ª tab = Resumen
<HubBody>
Cuerpo · aside opcional decide Layout A/B.
<HubBody aside={<Contexto/>}>
  {contenido}
</HubBody>  // sin aside = full
bg-slate-50/30 · p-3 sm:p-4 md:p-6 · grid 3 (con aside) o full · DENTRO del shell
