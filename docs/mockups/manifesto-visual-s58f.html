01 · Filosofía

Principios fundadores #

Cada decisión visual responde a uno de estos 6 principios. Si una pantalla no respeta alguno, debe corregirse.

P1

Densidad informativa con jerarquía clara

No esconder datos detrás de clicks. Mostrar lo relevante de un vistazo, con tamaños y pesos que guíen el ojo. Como Bloomberg Terminal pero hermoso.

P2

Banking-grade restraint

Los números dicen la verdad. Sin gradientes flashy, sin emojis decorativos, sin shadow excesivo. Mercury / Stripe Atlas.

P3

Color con propósito semántico

Teal = marca y acciones primarias. Emerald = positivo. Rose = negativo. Amber = atención. Slate = neutro. Nada decorativo.

P4

Tabular-nums siempre

Cualquier número (saldo, monto, porcentaje, contador) usa font-feature-settings: "tnum". Alineación vertical perfecta.

P5

Micro-interacciones sutiles

Hover ligero (translate-y -1px + shadow), focus ring teal, transiciones 150-200ms cubic-bezier. Nunca saltos bruscos.

P6

Cada pantalla declara su propósito

Header con título + subtítulo descriptivo. El usuario nunca duda de para qué es la pantalla actual.

07 · Botones

Sistema de botones #

5 variantes. La jerarquía visual indica la importancia de la acción.

Primary · acción dominante

Una por pantalla. La acción principal que el usuario debe hacer.

Primary soft · acción semántica

Acciones contextuales con color semántico (cobro=verde, pago=rojo, etc).

Outline · acción secundaria

Acciones útiles pero no la principal. Exportar, importar, filtrar.

Ghost · acción utilitaria

Refresh, ayuda, ajustes menores. Aparece pero no compite por atención.

Danger · acción destructiva

Solo aparece dentro de modal de confirmación. Nunca como acción primaria de pantalla.

Animaciones y feedback #

Sutiles, rápidas, intencionales. Comunican estado sin distraer.

Accesibilidad. Aparece al tab por teclado o focus programático.

Pulse soft · alertas urgentes

Para badges de notificación que requieren atención inmediata.

Slide-up · entrada de elementos

Cuando aparece nuevo contenido (toast, panel, sub-row), entra desde abajo con fade.

@keyframes slide-up { from { opacity: 0; transform: translateY(8px); } to { opacity: 1; transform: translateY(0); } } animation: slide-up 0.3s ease-out;

Loading skeleton · placeholder con shimmer

Cuando datos están cargando, mostrar la silueta del contenido. Nunca spinner solo.
