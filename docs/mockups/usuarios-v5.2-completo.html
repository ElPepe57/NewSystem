8.3Máquina de estados · ciclo de vida del usuario
invitado_no_registrado
Admin envió email · user no entró aún
pendiente_aprobacion
User completó signup · admin debe aprobar
activo
User puede operar normalmente
suspendido
Bloqueado temporal · admin lo reactiva
archivado
Soft-delete · 90 días en logs
Transiciones permitidas: invitado→activo (post-aprobación) · self-signup→pendiente_aprobacion → activo (post-aprobación admin) · activo↔suspendido (admin) · activo/suspendido→archivado (admin · soft-delete · typed-confirm)
Cross-references entre módulos
Inversionistas (Socios)
socioService.getAll() · users con rol='socio' compuestos con datosSocio · 1 sola fuente de verdad
Planilla
planillaService.getAll() · users con rol='planilla' compuestos con datosLaborales · sueldos · contratos · ausencias
Auditoría
auditLog · cada acción del user queda registrada con uid · IP · timestamp · módulo
