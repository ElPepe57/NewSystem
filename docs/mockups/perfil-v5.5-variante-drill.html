Concepto · qué cambia vs v5.4

Diferencia clave entre las 3 opciones que evaluamos

v5.4 actual Opción A · 3 tabs fijos

Todo el contenido vive en los 3 tabs · empleado tiene mucho material amontonado en Resumen.

Navegación simple
Consistente entre users
Resumen sobrecargado
Difícil drill profundo
descartada Opción B · tabs dinámicos

Tabs aparecen/desaparecen según rol · empleado ve "Mi planilla" tab nuevo.

Cada cosa en su lugar
Inconsistencia user a user
Multi-rol = 5+ tabs
Recomendada
v5.5 esta variante Opción C · drill-down

3 tabs fijos · pero quick actions abren sub-páginas dedicadas `/perfil/{seccion}` con back link.

Resumen limpio · sin overflow
Drill profundo cuando se necesita
Tabs fijos · consistencia
Multi-rol escala bien
Insight clave Opción C: diferencia "vista PERSONAL del dato" (vive en `/perfil/mi-planilla`) de "módulo de gestión" (vive en `/planilla`). Un empleado nunca debería entrar a `/planilla` directamente porque es la vista admin de la nómina de todos. Su vista PERSONAL vive en `/perfil/mi-planilla`.

Cross-link routing canon
• Quick action card (verde · sky · violet) → navega a /perfil/mi-X (sub-página propia)
• Cross-link CTA grande (banner gradient) → navega al MÓDULO completo (/planilla, /inversionistas)
• Back link en sub-página → vuelve a /perfil?tab=resumen

Mobile · UX patterns catalog · transitions + accessibility

Detalles que hacen la experiencia banking-grade en mobile

Touch targets ≥44×44px

Apple HIG · Material Design · todos los buttons y cards clickables.

w-11 h-11 (back arrow) min-h-[44px] (tabs scroll-x) min-h-[56px] (cross-link cards) min-h-[76px] (quick action cards)
Page transitions

Slide-from-right cuando navega a sub-página · slide-to-right en back.

transition: transform 250ms cubic-bezier(0.4, 0, 0.2, 1) forward: translateX(100% → 0) back: translateX(0 → 100%)
Scroll restoration

Volver al perfil restaura el scroll exacto donde estaba.

React Router scroll restoration plugin + Browser History API + scrollY saved en sessionStorage
Safe area inset (notch + home indicator)

Respeto del área segura · header + footer bottom-sheet.

padding-top: env(safe-area-inset-top) padding-bottom: env(safe-area-inset-bottom)
