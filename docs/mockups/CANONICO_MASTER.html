  • Gradientes pesados de fondo

⚠ Scrollbars · OCULTAR en mobile

Los scrollbars nativos del navegador rompen el look banking-grade. En cualquier elemento con overflow-x-auto en mobile (pills · tabs · KPIs scroll · sub-tabs) aplicar la utility: 

.scrollbar-hide {
  scrollbar-width: none;        /* Firefox */
  -ms-overflow-style: none;     /* IE/Edge */
}
.scrollbar-hide::-webkit-scrollbar {
  display: none;                /* Chrome/Safari/Opera */
}

La funcionalidad de scroll se mantiene · solo se oculta el chrome visual. Aplicar también via plugin Tailwind o en globals.css.

⛔ Anti-pattern · NO usar progressive disclosure

Esconder columnas con hidden md:block en una tabla NO es la solución. Genera:

  • Pérdida de información crítica en mobile (un usuario en mobile NO debería ver MENOS, debería ver lo MISMO en otro formato)
  • Touch targets muy chicos · checkboxes y botones diminutos
  • Scroll horizontal mental para entender · no respeta el modo de uso real
  • Estética banking-grade rota · "tabla apretada" se ve mal
  • Ninguna gran empresa moderna lo hace
