Notas de diseno — Wizard Envio Unificado

Por que 5 pasos fijos para todos los tipos

El stepper siempre muestra "Tipo / Origen / Destino / Logistica / Confirmar". Esto permite al usuario saber en todo momento en que parte del proceso esta, independientemente del tipo de envio elegido. El contenido interno de cada paso cambia, pero la estructura de navegacion es identica.

El Paso 1 como reemplazo del NuevoEnvioMenu

El dropdown actual no explica cuando usar cada tipo. La tarjeta del Paso 1 tiene nombre descriptivo + caso de uso tipico + indicadores visuales (color de grupo, icono de ruta). El usuario aprende mientras elige, no antes.

Tipos A, B, D — visibles pero no seleccionables

Se muestran en una seccion colapsable "Como nacen los envios A, B y D" para educar al usuario sin confundirlo. La seccion esta colapsada por default para no contaminar el flujo principal con opciones que el usuario no puede elegir.

Asimetria de picking (Pasos 2 vs 3)

En Caso C el picking ocurre en Paso 3 (porque en Paso 2 solo se elige la casilla). En Casos E, I, J el picking ocurre en Paso 2 (porque origen y picking van juntos). En Caso F el picking ocurre en Paso 3 (porque primero se elige la venta). Esta asimetria es correcta — forzar picking siempre en el mismo paso crearia relleno artificial.

Colores de grupo en el Paso 1

  • Teal: importaciones (C, J) — el color primario del sistema
  • Amber: operaciones Peru (E, F) — operaciones locales
  • Violet: casos especiales (G, I) — poco frecuentes
  • Slate con opacidad: automaticos (A, B, D) — informativos

Panel lateral persistente

El panel lateral se actualiza en tiempo real desde el Paso 1. Al elegir el tipo, ya muestra la ruta logistica esquematica. Al elegir la casilla en Paso 2, ya muestra el origen con bandera y nombre. Al definir costos en Paso 4, ya muestra el total landed con conversion a PEN. En el Paso 5 de confirmacion, el panel es un resumen compacto de todo.

Etiqueta del boton final por tipo

Caso C → "Crear y despachar →"
Caso J → "Crear envio J →"
Caso E → "Crear traslado →"
Caso F → "Crear despacho →"
Caso G → "Crear retorno fisico →"
Caso I → "Crear envio a tercero (bloquear stock) →"
