VISUAL-DECISION-1 · Matriz 2 ejes activos
Grilla 2D tipo Stripe: filas = tamaños, columnas = sabores. Celdas clickables (checkbox + label). Celdas grises = hueco (combinación que no existe). La excepción F4 "catálogo denso" aplica: la grilla ES el catálogo de variantes a crear. Permite ver todo el universo de combinaciones de un vistazo y marcar solo las que existen físicamente.
VISUAL-DECISION-2 · Tabla inline vs acordeón
Tabla inline compacta (F4 excepción "catálogo denso"): una fila por variante, columnas: SKU tentativo, contenido, sabor (si aplica), UPC, peso, chip duración, estrella principal, eliminar. Inline editable por celda. Más escaneable y eficiente que acordeones para 3-12 variantes. El chip duracion aparece solo para SUP.

Decisiones visuales estados F/G/H
VISUAL-DECISION-3 · Modo distintas presentaciones
3 botones radio para seleccionar el modo de variación. Cuando se activa "Distintas presentaciones": (a) cada fila obtiene un select de unidad propio, (b) se agrega la columna "Servings/día" como campo editable por fila, (c) el eje Sabor se desactiva con banner explicativo, (d) el ChipDuracion se calcula con la unidad propia de cada fila. Exclusivo con modo grilla 2D.
VISUAL-DECISION-4 · ChipDuracion con unidad continua
Cuando la unidad es continua (g/ml/lb), el chip de duración no puede calcular días exactos sin saber el dosaje por porción. Decisión: chip "pendiente" en gris con icono reloj y texto "100 g ÷ ?" + nota al pie "Ingresar dosaje en Sec. 2 para calcular". Evita mostrar un número erróneo y guía al usuario hacia dónde completar el dato faltante.
VISUAL-DECISION-5 · Marketing SKC sin disclaimer SUP
El banner de compliance INDECOPI difiere por línea: SKC muestra solo la nota cosmética ("fortalece la barrera" es válido). SUP agrega el disclaimer DIGEMID automático ("No reemplaza dieta balanceada · consulte a su médico"). El texto del compliance se adapta en el componente según lineaCodigo recibido por prop.
