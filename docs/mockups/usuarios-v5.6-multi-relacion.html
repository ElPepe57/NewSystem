Modelo de datos · colección relacionesLaborales

Cada relación es un documento separado · N docs por user · vigente/pausada/finalizada

v5.5 simple (limitado) 

{/* UserProfile */}
{
  uid: 'abc123',
  tipoRelacion: 'empleado',  // ❌ 1 solo valor
  /* sub-doc datosLaborales */
  /* sub-doc datosSocio */
}

❌ Si pasa de honorarios a empleado:
   se sobrescribe · pierde histórico
❌ Si es empleado Y socio:
   no se puede modelar bien
❌ Sin fechas de transición

v5.6 multi-relación 

{/* UserProfile */}
{ uid: 'abc123' /* sin tipoRelacion */ }

{/* relacionesLaborales/rel-001 */}
{
  userId: 'abc123',
  tipo: 'empleado',
  estado: 'vigente',
  fechaInicio: '2024-03-15',
  fechaFin: null,
  datosLaboralesSnapshot: { sueldo: 4500, ... }
}

{/* relacionesLaborales/rel-002 */}
{
  userId: 'abc123',
  tipo: 'socio',
  estado: 'vigente',
  fechaInicio: '2024-03-15',
  datosSocioSnapshot: { porcentaje: 45 }
}

Estados de una relación

vigente
Relación activa actualmente · cobra · acceso completo
pausada
Maternidad · estudios · licencia · NO cobra · mantiene cuenta
prueba
Período prueba · 3 meses iniciales · transición auto a vigente
finalizada
Cerrada · histórico inmutable · snapshot preservado
Snapshots inmutables: al finalizar una relación, el campo `datosLaboralesSnapshot` queda congelado con los datos exactos al momento de cierre. Si después el sueldo de la nueva relación cambia, el histórico mantiene los valores reales.

Chips dinámicos por relación
Vigentes: chip con check ✓ · Pausadas: chip con ⏸ amber · Prueba: chip con 🔵 + tiempo restante · Históricas: chip slate con icon history · MULTI: badge purple ★ si tiene ≥2 vigentes · RECLASIFICADA: badge si la actual viene de un cambio · RE-INCORPORADO: badge amber si tiene histórico finalizado del mismo tipo.

Flow "Pausar / Reanudar" · licencia

Maternidad · estudios · licencia sin goce · mantiene relación pero NO cobra

Pausar relación

Lucía Mendoza · Empleado · Vendedora

Una relación pausada:
  • Mantiene la cuenta del usuario activa
  • NO recibe sueldo · boletas se omiten
  • NO acumula vacaciones ni gratificación proporcional
  • Aparece en la lista con chip "PAUSADA" amber
  • Al reanudar · se calcula la fecha de retorno
