ACTO 1 · Bloque A

Gap analysis · /signup actual vs lo que necesitamos para externos

/signup existe pero está orientado al STAFF interno · externos necesitan flujo separado.

Aspecto /signup actual (staff) /solicitar-acceso (externos · NUEVO)
Audiencia Staff interno · empleados nuevos Contactos de proveedores · clientes B2B · transportistas · colaboradores
Campos Nombre · email · password + Empresa · cargo · motivo · buscador Maestros · referencia
Vinculación previa — (no aplica) Sugerencia auto-match dominio email → Maestro
Estado post-submit UserProfile.estado='pendiente_aprobacion' SolicitudAcceso.estado='pendiente' · NO crea User aún
Visibilidad admin Lista usuarios filtro=pendiente Bandeja propia con badge "🔔 N solicitudes"
Aprobación AprobarUsuarioModal · 1 click AprobarUsuarioModal extendido · revisa Maestro vinculado + roles
Roles sugeridos Vendedor · comprador · almacenero · etc Solo "invitado" (rol limitado · solo ve lo que admin habilita)
Protección anti-spam Asumida internal · sin reCAPTCHA reCAPTCHA v3 + rate limit por IP (es público en internet)
⚠️ Decisión arquitectónica

Las dos rutas NO se fusionan · son flujos distintos para audiencias distintas. /signup queda para staff (admin lo usa pocas veces · prefiere invitar). /solicitar-acceso es para externos auto-anunciándose · admin lo procesa con más controles.

Extensión approveUser CF · acepta solicitud
// CF: aprobarSolicitudAcceso
// (extensión de approveUser actual)
interface AprobarSolicitudInput {
  solicitudId: string;
  rolesAsignados: UserRole[];

  // ✨ NUEVO · vinculación Maestro
  entidadMaestroRef?: {
    tipo: 'cliente' | 'proveedor'
        | 'marca';
    id: string;
    rolEnEntidad?: string;
  };

  // Mensaje opcional para el invitado
  mensajePersonalizado?: string;
}

// Server logic atómico (Firestore txn):
async function aprobarSolicitudAcceso(input) {
  // 1. Lee solicitud
  const sol = await getSolicitud(input.id);
  if (sol.estado !== 'pendiente') throw;

  // 2. Crea UserProfile (estado='pendiente_activacion')
  const uid = await createUser({
    email: sol.email,
    displayName: sol.nombreCompleto,
    origen: 'self_signup',
    rolesAsignados: input.rolesAsignados,
  });

  // 3. Crea RelacionLaboral con vinculación
  await createRelacion({
    userId: uid,
    tipo: 'externo',
    estado: 'vigente',
    subTipo: derivarSubTipo(sol.tipoRelacion),
    entidadMaestroRef: input.entidadMaestroRef,
  });

  // 4. Crea Invitacion (token + email Resend)
  const inv = await crearInvitacion(uid);

  // 5. Update solicitud
  await updateSolicitud(input.id, {
    estado: 'aprobada',
    uidUsuarioCreado: uid,
    invitacionId: inv.id,
  });

  return { uid, invitacionId: inv.id };
}
