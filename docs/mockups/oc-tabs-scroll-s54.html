Barra de tabs con scroll horizontal · 5 alternativas modernas

Problema actual: con 9 tabs, el contenedor activa el scrollbar oscuro nativo. Abajo hay 5 alternativas modernas que lo eliminan.

🔎 Estado actual — scrollbar oscuro visible
⚠️ Scrollbar nativo grueso que rompe la estética.
V1 ⭐ RECOMENDADA

Scroll invisible + fade degradado a la derecha

Scrollbar oculto + degradado sutil a la derecha indica "hay más contenido". Patrón usado por Notion, Linear, Vercel, Raycast.

✓ Minimalista · 0 UI chrome extra · responsive natural
✗ Algunos usuarios pueden no descubrir scroll horizontal (el fade mitiga)
V2

Fade + flechas de navegación discretas

V1 + botón-flecha circular en el extremo. Aparece solo cuando hay overflow oculto. Patrón usado por Stripe, Figma, Chrome tabs.

✓ Descubribilidad explícita (flechas clickables) · soporta teclado
✗ Un poco más de UI chrome · requiere lógica JS para mostrar/ocultar flechas
V3

Tabs visibles + dropdown "Más ▾" con el resto

Muestra solo los tabs que caben; el resto va en un menú "Más". Clásico de Odoo, Zoho, Google Workspace, Gmail toolbar.

✓ Máxima limpieza visual · sin scroll · clásico conocido · teclado-friendly
✗ Requiere medir qué tabs caben en runtime · los tabs "ocultos" pueden sentirse de 2da categoría
V4

Wrap a 2 líneas (todos siempre visibles)

Los tabs se envuelven cuando no caben. Patrón de GitHub repo header, Jira.

✓ Todos los tabs siempre visibles · 0 interacción requerida
✗ Altura variable según viewport · la línea del tab activo se rompe si está en la 1ra fila
V5

Sidebar vertical (tabs como menú lateral izquierdo)

Tabs como menú vertical a la izquierda. Patrón Notion settings, Linear issue detail.

Contenido del tab activo (Productos)
✓ Escalable a 15+ tabs · jerarquía clara · lectura vertical natural
✗ Cambia la estructura del modal a 2 columnas · reduce ancho del contenido · pierde consistencia con detalle Envío

🎯 Recomendación

V1 (fade invisible) — la más simple y elegante. Funciona con trackpad / shift+rueda / drag. Patrón moderno estándar. Mi recomendación.

V2 (fade + flechas) — agrega descubribilidad con flechas discretas. Muy sólida si te importa la accesibilidad con ratón.

V3 (Más ▾) — look "desktop clásico" más que "app moderna". Mejor si quieres una estética ERP tradicional.

V4 (wrap) — mantiene todo visible pero se siente cargado con 9 tabs.

V5 (sidebar) — mejor si el modal crece a 15+ tabs. Hoy es overkill y rompe consistencia con Envío.

Propuesta combinada ⭐: implementar V1 + ingrediente de V2 — fade+scroll invisible por default, flechitas que aparecen solo cuando el contenedor detecta overflow oculto. Lo mejor de ambos: limpio siempre, descubrible cuando hace falta.
