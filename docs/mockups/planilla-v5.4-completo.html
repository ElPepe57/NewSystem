4 tipos de esquema · ejemplos visuales con configuración

Casos reales para Vita Skin · cómo se configura cada uno

A

TIPO A · Comisión sobre ventas

Vendedor recibe % sobre lo facturado. Puede ser lineal (% fijo) o escalonado (% sube con monto).

Ejemplo: Comisión escalonada
· Hasta S/ 50K: 2% comisión
· S/ 50-100K: 4% comisión
· Sobre 100K: 6% comisión
Pull data de: /ventas · filtro vendedorId === uid
B

TIPO B · Bono por meta

Equipos logística/compras reciben bono si cumplen métrica grupal del período.

Ejemplo: OC en tiempo
· Métrica: OC recibidas a tiempo · meta >95%
· Bono base: S/ 800 dividido equipo
· Bonus: +50% si >98%
Pull data de: /ordenesCompra · estado · fechaRecepcion vs esperada
C

TIPO C · Bono por KPI

Roles funcionales (finanzas · gerente) reciben bono si KPI específico se mantiene saludable.

Ejemplo: Cartera vencida < 5%
· Aplica: rol finanzas
· Métrica: % cartera vencida vs total
· Bono mensual si cumple: S/ 500
Pull data de: /finanzas/cuentasCorrientes · aging buckets
D

TIPO D · Bono fijo

Pago puntual o recurrente sin métrica calculada. Bono navidad · bono performance anual · etc.

Ejemplo: Bono navidad gerencia
· Aplica: usuarios específicos (gerente)
· Monto: S/ 5,000
· Frecuencia: anual · diciembre
No requiere pull · monto fijo configurado

Principio: NO se duplica data

Planilla siempre pulls fresh de los módulos cuando necesita métricas (comisiones · bonos meta). Esto garantiza consistencia · zero divergencia · y al cambiar una venta · el cálculo siguiente refleja el cambio.
